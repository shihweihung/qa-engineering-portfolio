# Automation

My honest relationship with test automation: I've participated in automation strategy and framework evaluation discussions throughout my career, and I'm actively building hands-on skills.

---

## What I've Actually Done

My practical automation experience is in **API testing** — using Postman for manual API exploration and verification, and doing basic scripted checks during feature validation and issue troubleshooting. This is the area where I have real hands-on experience.

For broader automation (mobile UI, web UI, full pytest frameworks), my role has been more on the **strategy and evaluation side**: participating in discussions about what to automate, evaluating framework options, and understanding tradeoffs — rather than being the primary implementer.

---

## What I'm Currently Learning

I'm actively learning test automation through hands-on practice, using AI tools to help me understand code I'm writing and get feedback on implementation decisions. Current areas:

This is deliberate skill development, not experience I've shipped in production.

## Why I'm Investing in This
